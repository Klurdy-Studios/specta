# Epic 009 — Skills & Native Agent Surfaces

## Goal

Generate native Specta experiences inside supported coding agents.

## Description

Skills and native commands are generated from Workflow Definitions using the
conventions of their target coding agent. Specta remains agent-agnostic: business
logic stays in Specta Core and workflow orchestration stays in the Workflow
Engine. MCP is a platform API, not the primary interface.

## User Stories

- Generate Codex Skills.
- Install Codex Skills into the active project Skill directory during init.
- Configure a local CLI helper for development testing before publication.
- Generate Claude Skills.
- Generate Cursor Commands.
- Generate VS Code Commands.
- Support future native agent surfaces.
- Generate workflow commands from Workflow Definitions.

## Deliverables

- Skill Generator
- Workflow Manifest
- Prompt Template Library
- Workflow Commands
- Codex Skill installer
- Runtime CLI helper configuration
- MCP platform capabilities

## Acceptance Criteria

- Supported Skills expose native Specta workflows.
- Workflow commands are discoverable in the target coding agent.
- Codex Skills are installed into `.codex/skills/` and invoke the configured
  local or published CLI helper internally.
- Skills do not duplicate business logic or workflow orchestration.
- MCP capabilities remain available where supported.
