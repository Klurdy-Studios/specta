# Epic 009 — Agent Integrations

## Goal

Provide native Specta experiences inside supported coding agents.

## Description

Each integration exposes Specta workflows using the conventions of its target
coding agent. Specta remains agent-agnostic: business logic stays in Specta Core
and workflow orchestration stays in the Workflow Engine. MCP is one integration
mechanism, not the primary interface.

## User Stories

- Provide Codex integration.
- Provide Claude Code integration.
- Provide Cursor integration.
- Provide VS Code integration.
- Support future agent integrations.
- Expose native workflow commands.
- Support the agent adapter framework.

## Deliverables

- Codex Connector
- Agent Adapter API
- Prompt Template Library
- Workflow Commands
- MCP capabilities as an integration mechanism

## Acceptance Criteria

- Supported integrations expose native Specta workflows.
- Workflow commands are discoverable in the target coding agent.
- Agent Integrations do not duplicate business logic or workflow orchestration.
- MCP capabilities remain available where supported.
